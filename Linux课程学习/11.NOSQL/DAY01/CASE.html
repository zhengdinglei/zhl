6）设置变量test，值为123，查看变量test的值

常用指令操作：

set keyname keyvalue 存储

get keyname 获取

copytextpop-up
  1. 127.0.0.1:6379> set test 123
  2. OK
  3. 127.0.0.1:6379> get test
  4. "123"
127.0.0.1:6379> set test 123
OK
127.0.0.1:6379> get test
"123"

del keyname 删除变量

copytextpop-up
  1. 127.0.0.1:6379> set k1 v1
  2. OK
  3. 127.0.0.1:6379> get k1
  4. "v1"
  5. 127.0.0.1:6379>  del k1
  6. (integer) 1
127.0.0.1:6379> set k1 v1
OK
127.0.0.1:6379> get k1
"v1"
127.0.0.1:6379>  del k1
(integer) 1

keys * 打印所有变量

copytextpop-up
  1. 127.0.0.1:6379> keys *
  2. 1) "test"
127.0.0.1:6379> keys *
1) "test"

EXISTS keyname 测试是否存在

copytextpop-up
  1. 127.0.0.1:6379> exists  k1
  2. (integer) 0
127.0.0.1:6379> exists  k1
(integer) 0

type keyname 查看类型

copytextpop-up
  1. 127.0.0.1:6379> set k2 v1
  2. OK
  3. 127.0.0.1:6379> type k2
  4. string
127.0.0.1:6379> set k2 v1
OK
127.0.0.1:6379> type k2
string

move keyname dbname 移动变量

copytextpop-up
  1. 127.0.0.1:6379> move k2 1            //移动k2到1库
  2. (integer) 1
127.0.0.1:6379> move k2 1			//移动k2到1库
(integer) 1

select 数据库编号0-15 切换库

copytextpop-up
  1. 127.0.0.1:6379> select 1        //切换到1库
  2. OK
  3. 127.0.0.1:6379[1]> keys *            //查看有k2

127.0.0.1:6379> select 1		//切换到1库
OK
127.0.0.1:6379[1]> keys *			//查看有k2

expire keyname 10 设置有效时间

copytextpop-up

  2. (integer) 1
127.0.0.1:6379[1]> EXPIRE k2 10
(integer) 1

ttl keyname 查看生存时间

copytextpop-up
  1. 127.0.0.1:6379[1]> ttl k2
127.0.0.1:6379[1]> ttl k2

flushall 删除所有变量

copytextpop-up
  1. 127.0.0.1:6379[1]>  FLUSHALL
  2. OK
127.0.0.1:6379[1]>  FLUSHALL
OK

save 保存所有变量

copytextpop-up
  1. 127.0.0.1:6379[1]> save
  2. OK
127.0.0.1:6379[1]> save
OK

shutdown 关闭redis服务

2 案例2：修改Redis服务运行参数

2.1 问题

  • 具体要求如下：
  • 端口号 6351
  • IP地址 192.168.4.51
  • 连接密码 123456
  • 客户端连接Redis服务

2.2 步骤

实现此案例需要按照如下步骤进行。

步骤一：修改redis运行参数

1）

copytextpop-up
  1. [root@redis1 utils]# cp /etc/redis/6379.conf  /root/6379.conf     
  2. //可以先备份一份，防止修改错误没法还原
  3. [root@redis1 utils]# /etc/init.d/redis_6379 stop
  4. [root@redis1 utils]# vim /etc/redis/6379.conf
  5. ...
  6. bind  192.168.4.51                //设置服务使用的ip
  7. port 6351                            //更改端口号
  8. requirepass 123456                //设置密码
  9. [root@redis1 utils]# /etc/init.d/redis_6379 start
  10. Starting Redis server...
  11. [root@redis1 utils]# ss -antul | grep 6351        //查看有端口6351
  12. tcp    LISTEN     0      128    192.168.4.51:6351                  *:*
[root@redis1 utils]# cp /etc/redis/6379.conf  /root/6379.conf 	
//可以先备份一份，防止修改错误没法还原
[root@redis1 utils]# /etc/init.d/redis_6379 stop
[root@redis1 utils]# vim /etc/redis/6379.conf
...
bind  192.168.4.51				//设置服务使用的ip
port 6351							//更改端口号
requirepass 123456				//设置密码
[root@redis1 utils]# /etc/init.d/redis_6379 start
Starting Redis server...
[root@redis1 utils]# ss -antul | grep 6351		//查看有端口6351
tcp    LISTEN     0      128    192.168.4.51:6351                  *:*

由于修改了配置文件所以在连接的时候需要加上ip和端口

copytextpop-up
  1. [root@redis1 utils]# redis-cli  -h 192.168.4.51 -p 6351 
  2. 192.168.4.51:6351> ping
  3. (error) NOAUTH Authentication required.
  4. 192.168.4.51:6351> auth 123456            //输入密码才能操作（因为之前设置过密码）
  5. OK
  6. 192.168.4.51:6351> ping
  7. PONG
[root@redis1 utils]# redis-cli  -h 192.168.4.51 -p 6351 
192.168.4.51:6351> ping
(error) NOAUTH Authentication required.
192.168.4.51:6351> auth 123456			//输入密码才能操作（因为之前设置过密码）
OK
192.168.4.51:6351> ping
PONG

还可以直接在命令行输入密码连接

copytextpop-up
  1. [root@redis1 utils]# redis-cli  -h 192.168.4.51 -p 6351  -a 123456
  2. 192.168.4.51:6351> ping
  3. PONG
[root@redis1 utils]# redis-cli  -h 192.168.4.51 -p 6351  -a 123456
192.168.4.51:6351> ping
PONG
